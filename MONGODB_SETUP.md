# 🍃 MongoDB Atlas Setup Guide

This guide will help you set up a free MongoDB Atlas database for your AI Health Monitoring application.

## 🎯 Why MongoDB Atlas?

- **Free Tier**: 512 MB storage (perfect for starting)
- **Cloud-based**: No local installation needed
- **Easy Setup**: Get running in 5 minutes
- **Scalable**: Grow when you need to

## 📝 Step-by-Step Setup

### Step 1: Create MongoDB Atlas Account

1. Go to https://www.mongodb.com/cloud/atlas
2. Click **"Try Free"** or **"Register"**
3. Sign up with:
   - Email address
   - Password
   - Or use Google/GitHub account

### Step 2: Create Your Free Cluster

1. After logging in, click **"Build a Database"**
2. Select **"FREE"** tier (M0 Sandbox - 512 MB)
3. **Configure Cluster**:
   - **Cloud Provider**: AWS (recommended)
   - **Region**: Choose closest to your users
     - US East (Virginia) - for US users
     - Europe (Ireland) - for EU users
     - Asia Pacific (Singapore) - for Asia users
4. **Cluster Name**: Enter "healthdb" (or your preferred name)
5. Click **"Create Cluster"**

### Step 3: Create Database User

1. Click **"Database Access"** in the left sidebar
2. Click **"Add New Database User"**
3. Fill in the form:
   - **Username**: `healthapp` (or your preferred username)
   - **Password**: Click "Autogenerate Secure Password"
   - **IMPORTANT**: Copy and save this password! You'll need it for the connection string
4. **Privileges**: Leave as "Read and write to any database"
5. Click **"Create User"**

### Step 4: Whititelist IP Addresses

1. Click **"Network Access"** in the left sidebar
2. Click **"Add IP Address"**
3. Choose **"Allow Access from Anywhere"** (0.0.0.0/0)
   - This allows your Vercel app to connect
4. Click **"Confirm"**

### Step 5: Get Your Connection String

1. Click **"Database"** in the left sidebar
2. Find your cluster and click **"Connect"**
3. Select **"Connect your application"**
4. **Driver**: Select **"Node.js"**
5. **Version**: Select latest (e.g., "5.5 or later")
6. Copy the connection string

It should look like:
```
mongodb+srv://healthapp:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

### Step 6: Replace Password in Connection String

Replace `<password>` with the actual password you generated in Step 3.

**Example**:
```
mongodb+srv://healthapp:abc123xyz789@cluster0.xxxxx.mongodb.net/healthdb?retryWrites=true&w=majority
```

### Step 7: Add Database Name (Optional but Recommended)

Add `/healthdb` before the `?` to specify the database name:

```
mongodb+srv://healthapp:abc123xyz789@cluster0.xxxxx.mongodb.net/healthdb?retryWrites=true&w=majority
```

## 🔧 Using Your Connection String

### For Vercel Deployment

1. Go to your Vercel project dashboard
2. Go to **Settings** → **Environment Variables**
3. Add a new variable:
   - **Key**: `MONGODB_URI`
   - **Value**: Paste your connection string (with password)
4. Click **Save**
5. Redeploy your app

### For Local Development

Create `health-backend/.env`:
```env
MONGODB_URI=mongodb+srv://healthapp:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/healthdb?retryWrites=true&w=majority
JWT_SECRET=your-secret-key-here
NODE_ENV=development
PORT=5000
```

## ✅ Testing Your Connection

### Test Connection String

You can test if your connection string works:

```bash
# Using MongoDB Shell (if installed)
mongosh "mongodb+srv://healthapp:YOUR_PASSWORD@cluster0.xxxxx.mongodb.net/healthdb?retryWrites=true&w=majority"
```

Or test it directly in your app by running:
```bash
cd health-backend
npm run dev
```

If you see "✅ MongoDB connected successfully", your connection works!

## 🔒 Security Best Practices

### DO:
- ✅ Use strong, autogenerated passwords
- ✅ Keep passwords secure (don't commit to git)
- ✅ Use environment variables
- ✅ Restrict database user permissions

### DON'T:
- ❌ Share your connection string publicly
- ❌ Commit connection strings to git
- ❌ Use default passwords
- ❌ Allow access from specific IPs unless needed

## 📊 Managing Your Database

### View Your Data

1. In MongoDB Atlas, go to **"Database"** → **"Browse Collections"**
2. You'll see your collections after users register
3. Collections you'll see:
   - `users` - User accounts
   - `healthmetrics` - Health data
   - `diseasepredictions` - AI predictions
   - `dietplans` - Meal plans
   - `yogasessions` - Yoga sessions

### Monitor Usage

1. Go to **"Metrics"** in MongoDB Atlas
2. Monitor:
   - Storage used (free tier: 512 MB)
   - Network traffic
   - Connection count

## 🐛 Troubleshooting

### Issue: "Connection timeout"
**Solution**:
- Check IP whitelist includes 0.0.0.0/0
- Verify cluster is running (not paused)
- Check firewall settings

### Issue: "Authentication failed"
**Solution**:
- Verify username and password are correct
- Check user has correct permissions
- Ensure password is URL-encoded if it has special characters

### Issue: "Cluster not found"
**Solution**:
- Verify cluster name in connection string
- Check cluster is created and active
- Wait a few minutes after cluster creation

## 💡 Tips

1. **Save Your Credentials**: Store connection strings securely
2. **Monitor Usage**: Check your free tier limits
3. **Backup Data**: MongoDB Atlas has automatic backups
4. **Scale Up**: When you exceed 512 MB, consider upgrading
